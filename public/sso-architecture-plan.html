SSO & Client Onboarding Architecture Plan

SERNET Group — Unified Identity & Business Integration

Document Version: 1.0  |  Prepared: February 2026  |  Classification: Internal

1. Executive Summary

This document outlines the architecture for a unified Single Sign-On (SSO) system across the SERNET Group ecosystem. The design establishes Azure AD B2C as the central Identity Hub and Zoho CRM as the Business Hub, enabling seamless client onboarding, cross-platform authentication, and centralised CRM management across all brand applications.

Core Principle

One identity, many experiences. A client onboards once via TickFunds and gains authenticated access to all SERNET platforms — investment management, education, insurance, and marketing resources.

3. Architecture Design

3.1 Identity Hub — Azure AD B2C

Azure AD B2C serves as the single source of truth for all user identities across the ecosystem. It was chosen because:

  • TickFunds (the primary onboarding app) is already Azure-based
  • Native OIDC/OAuth 2.0 support for all app types
  • Built-in MFA, conditional access, and custom user flows
  • Custom claims/attributes for role & tier management
  • Enterprise-grade compliance (SOC 2, ISO 27001)

3.2 Business Hub — Zoho CRM

Zoho CRM manages the business lifecycle of clients — not their authentication. It receives data downstream from TickFunds and tracks:

  • Lead → Client conversion pipeline
  • Product subscriptions & AUM tracking
  • Communication history & engagement scoring
  • Cross-sell/up-sell opportunities across brands

4. Implementation Phases

Phase 1 — Azure AD B2C Tenant Setup Foundation

Duration: 1–2 weeks  |  Priority: Critical

Deliverables

  • Create Azure AD B2C tenant (e.g., sernetgroup.onmicrosoft.com)
  • Configure custom attributes: client_type, membership_tier, kyc_status, onboarded_at, referral_code, app_access
  • Create user flows: Sign-up/Sign-in, Password Reset, Profile Edit
  • Register applications (one per brand app — TickFunds, Findemy, Tushil, SERNET)
  • Configure redirect URIs for each application
  • Set up MFA policy (SMS/Authenticator)
  • Configure token claims to include custom attributes
  • Test OIDC token issuance with Postman/curl

Phase 2 — TickFunds Native Integration Primary App

Duration: 2–3 weeks  |  Priority: Critical

Deliverables

  • Integrate MSAL.js / MSAL-React into TickFunds
  • Implement sign-up flow that captures: name, email, phone, PAN, client_type
  • On successful onboard → write custom attributes to B2C profile
  • Configure silent token refresh for seamless sessions
  • Implement role-based access within TickFunds using B2C claims
  • Add KYC status tracking (pending → verified → active)
  • Test full onboarding flow end-to-end

Phase 3 — TickFunds → Zoho CRM Sync Integration

Duration: 1–2 weeks  |  Priority: High

Deliverables

  • Create Zoho CRM OAuth app & generate refresh token
  • Build backend webhook/edge function: triggered on TickFunds onboarding
  • Map fields: B2C profile → Zoho Contact (name, email, phone, client_type, onboarded_at)
  • Implement upsert logic (create if new, update if existing — match on email)
  • Add Zoho custom fields: azure_b2c_oid, membership_tier, products_active[]
  • Set up Zoho automation: auto-assign to relationship manager based on client_type
  • Implement error handling & retry logic for failed syncs
  • Build sync status dashboard (optional: in SERNET admin panel)

Phase 4 — SERNET Website SSO Marketing Site

Duration: 1–2 weeks  |  Priority: Medium

Deliverables

  • Install OIDC library (e.g., oidc-client-ts or @azure/msal-browser)
  • Configure B2C app registration redirect URI for SERNET
  • Add "Sign In" button to header (redirect to B2C login page)
  • On token received → create/update local profile in Lovable Cloud DB
  • Implement gated content: premium articles, reports, tools for authenticated users
  • Show personalised dashboard: tier, benefits, cross-app links
  • Handle token refresh & logout (clear B2C session + local)
  • Preserve existing admin auth (internal) — separate from client SSO

Phase 5 — Findemy (WordPress) SSO LMS

Duration: 1–2 weeks  |  Priority: Medium

Deliverables

  • Install WordPress OIDC plugin (e.g., "OpenID Connect Generic" or "miniOrange")
  • Configure plugin with B2C tenant endpoints (authorize, token, userinfo, JWKS)
  • Map B2C claims to WordPress user fields (display name, email, role)
  • Configure auto-provisioning: create WP user on first B2C login
  • Map membership_tier → WP role (subscriber, student, premium_student)
  • Grant course access based on tier/client_type claims
  • Test SSO flow: SERNET → "Learn on Findemy" → auto-login → course access
  • Disable native WP registration (force B2C sign-up)

Phase 6 — Tushil (Insurance) SSO Insurance App

Duration: 2–3 weeks  |  Priority: Medium

Deliverables

  • Integrate MSAL or OIDC client into Tushil app
  • Configure B2C app registration for Tushil
  • Pre-populate insurance forms with B2C profile data (name, DOB, contact)
  • Sync insurance product data back to Zoho CRM contact record
  • Implement cross-sell triggers: if client has investment but no insurance → prompt
  • Test SSO flow: TickFunds → "Get Insurance" → auto-login on Tushil

Phase 7 — Unified Client Dashboard Advanced

Duration: 3–4 weeks  |  Priority: Low (future)

Deliverables

  • Build unified dashboard on SERNET (or standalone micro-app)
  • Show cross-platform status: investments, courses enrolled, insurance policies
  • Aggregate data from TickFunds API, Findemy API, Tushil API
  • Membership tier management: upgrade/downgrade with Razorpay/Stripe
  • Referral programme tracking & rewards
  • Notification centre (cross-app alerts)
  • Profile management synced back to B2C

6. Security Considerations

  • Token validation: Every app must validate JWT signature against B2C JWKS endpoint
  • Scope separation: Each app gets its own B2C app registration with specific scopes
  • Refresh tokens: Use rotating refresh tokens; max lifetime 24 hours
  • MFA: Enforced for HNW and Institutional clients; optional for Retail
  • CORS: Strict origin allowlisting per app domain
  • Webhook security: TickFunds → Zoho webhook signed with HMAC-SHA256
  • Data residency: B2C tenant in India region (Central India) for compliance
  • Audit logging: All auth events logged in Azure Monitor + forwarded to Zoho
